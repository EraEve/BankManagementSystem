## 1. 系统概述

银行智能管理系统是一个综合性的银行管理软件，完整实现了银行日常运营所需的全部核心功能模块。系统采用 **C++ 控制台应用** 作为核心业务引擎，辅以 **Python Flask + HTML5 前端** 提供 Web 访问界面。

### 1.1 核心特性

| 特性 | 说明 |
|------|------|
| **10大需求模块** | 职员管理、客户管理、银行卡管理、存取贷业务、业务查询、排队管理、网点导航、智能管理、身份核验、主界面 |
| **13种数据结构/算法** | 哈希表、链式队列、BST、邻接矩阵图、Dijkstra最短路径、冒泡排序、快速排序、二分查找、滑动窗口、加权多因子模型、ISO 7064校验、生物特征哈希、模拟OCR |
| **三级权限体系** | 管理员（全部功能）、职员（业务操作）、客户（自助查询） |
| **文件持久化** | 全部数据以管道分隔符文本文件存储，启动自动加载 |
| **双端支持** | C++控制台（核心）+ Web前端（Flask+Bootstrap 5） |
| **一键部署** | 支持Render云部署，本地双击bat即启动Web端 |

### 1.2 版本演进

| 版本 | 日期 | 内容 |
|------|------|------|
| v1.0 中期版 | 2025-05 | C++基础业务（登录/存取款/转账），单一文件743行 |
| v2.0 最终版 | 2025-06 | 完整10大需求，12个模块文件，~3500行 |

```bash

cd BankManagementSystem

# 2. 编译
g++ main.cpp -o bank_system -std=c++11
```

### 模块4: 存取贷业务管理

**菜单路径**: 管理员/职员登录 → [需求4] 存取贷业务管理

**权限**: 管理员和职员

**功能菜单**:

```
  ║  0. 返回上级菜单                ║
  ╚══════════════════════════════════╝
```

**各业务详细说明**:

#### 存款
1. 输入银行卡号
2. 输入存款金额（正数）
3. 系统校验：卡状态正常
4. 卡余额增加，客户金融资产同步增加
5. 自动记录交易（类型：存款）

#### 取款
1. 输入银行卡号 → 输入金额
2. **三重校验**: 卡状态正常 → 余额充足 → 不超过日累计限额
3. 取款失败时自动记录失败原因（余额不足/超限额）
4. 成功后余额和资产同步减少

#### 转账
1. 输入转出卡号 → 转入卡号 → 金额
2. **安全校验**: 两张卡均正常 → 不能同卡转账 → 转出卡余额充足
3. 转出卡减、转入卡增（原子操作）
4. 自动记录交易

#### 贷款（仅限信用卡）
1. 输入信用卡号 → 贷款金额
2. 校验：可用额度 = 信用额度 − 当前借贷余额
3. 贷款金额不得超过可用额度
4. 借贷余额增加，卡内余额增加（现金入账）

#### 还款（仅限信用卡）
1. 输入信用卡号 → 还款金额
2. 校验：还款金额不超过借贷余额，且不超过卡内余额
3. 借贷余额减少，卡内余额减少

#### 利息计算
- **日利息** = 本金 × (年利率%/100) / 365
- **月利息** = 本金 × (年利率%/100) / 12
- 储蓄卡/借记卡按余额计息，信用卡按借贷余额计息
- 支持单卡查看和全行批量计算

**数据结构**: `vector<Transaction>` 顺序表

**数据文件**: `data/transaction.txt`

```
  ║  4. 每日客流统计                ║
  ║  5. 历史排队记录                ║
  ║  0. 返回上级菜单                ║
  ╚══════════════════════════════════╝
```

**排队机制**:

```
       客户到达
           │
     ┌─────┴─────┐
     ▼           ▼
  VIP队列     普通队列
  (优先服务)   (顺序服务)
     │           │
     └─────┬─────┘
           ▼
    ┌──────────────┐
    │  VIP窗口(0)  │ ← 优先从VIP队列取
    │  普通窗口(1) │ ← 先查VIP队列，再查普通队列
    └──────────────┘
           │
           ▼
      办理完成 → 客户评分(1-5分)
           │
           ▼
      每日客流统计
```

**VIP优先策略**:
- VIP窗口: 先处理VIP队列，VIP队列空时才处理普通队列
- 普通窗口: 先查看VIP队列是否有等待（跨窗口服务），再处理普通队列

**取号流程**:
1. 职员输入客户编号
2. 选择业务类型（存款/取款/转账/贷款/开户/其他）
3. 系统自动生成票号（Q+4位数字）
4. 根据客户类型自动路由到VIP或普通队列
5. 显示当前排队人数

**叫号流程**:
1. 选择窗口类型（VIP窗口/普通窗口）
2. 系统按优先级自动出队
3. 显示客户信息 → 确认完成 → 客户评分
4. 业务未完成可重新入队

**每日客流统计**:
- 记录日期、总人数、VIP人数、普通人数、平均评分
- 自动累计当日数据
- 持久化到 `data/daily_stats.txt`

**核心数据结构**: `LinkedQueue<T>` 链式队列（自行实现，支持入队/出队/遍历/条件删除）

**数据文件**: `data/queue.txt`, `data/daily_stats.txt`

- 超过5笔/小时触发报警

- 信用评分均值和标准差
- 信用评级分布（AAA/AA/A/B/C）

**信用评级模型**（加权多因子）:

| 因子 | 权重 | 说明 |
|------|------|------|
| 金融资产 | 30% | 资产越多评分越高 |
| 交易活跃度 | 20% | 正常交易频率 |
| 还款记录 | 25% | 按时还款加分 |
| 负债率 | 25% | 负相关（负债越高扣分越多） |

**五级评级**:

| 评分范围 | 等级 | 含义 |
|---------|------|------|
| 900+ | AAA | 极优 |
| 800-899 | AA | 优秀 |
| 700-799 | A | 良好 |
| 600-699 | B | 一般 |
| <600 | C | 需关注 |

#### 8.4 风控审批

综合评估维度:
1. 客户信用评分
2. 负债率（总贷款 / (总余额 + 总信用额度)）
3. 负债率 > 50%: 扣100分；负债率 > 80%: 扣200分
4. 金融资产 > 10万: +50分；> 50万: 再+50分

**审批结论**:
- 风控分数 ≥ 700: ✓ 通过（低风险）
- 风控分数 ≥ 600: ⚡ 条件通过（中风险，建议降低额度）
- 风控分数 < 600: ✗ 拒绝（高风险）

| 风险偏好 | 触发条件 | 推荐配置 |

| 稳健型 | 信用分 > 600 | 债券基金35% + 定存35% + 货币基金15% + 活期15% |
| 保守型 | 信用分 ≤ 600 | 定存50% + 货币基金25% + 活期25% |

#### 9.1 身份证号码校验（菜单选项 1）

**输入**: 18位中国居民身份证号码

**校验流程**:
1. 长度检查: 必须18位
2. 数字检查: 前17位必须为数字
3. 末位检查: 数字或X
4. 出生日期校验: 年(1900-2100)、月(1-12)、日(1-31)
5. ISO 7064:1983 MOD 11-2 校验位计算

**校验位算法**:
```
权重: [7,9,10,5,8,4,2,1,6,3,7,9,10,5,8,4,2]
Sum = Σ(第i位数字 × 权重[i])
校验码映射 = ['1','0','X','9','8','7','6','5','4','3','2']
expected = 校验码映射[Sum % 11]
```

**输出报告**: 行政区划（省份）、出生日期、性别、年龄、校验结果

**评分标准**:
- 身份证合法: 40分
- 姓名已录入: 20分
- 电话已录入: 15分
- 地址已录入: 10分

#### 9.4 生物特征录入（菜单选项 4）

**模拟采集流程**:

#### 9.5 生物特征验证（菜单选项 5）

2. 与已存储模板进行相似度比对
3. 输出匹配报告

**匹配阈值**:

- 双模态: 90%

**验证结论**:
- ≥阈值: ✓ 通过（确认为本人）
- 50% ~ 阈值: ⚡ 相似度偏低，需人工复核
- < 50%: ✗ 不匹配，验证失败

## 10. 常见问题

### Q1: 编译报错 "g++: command not found"

**解决**: 安装MinGW-w64或GCC编译器。
- Windows: 下载 MinGW-w64，添加 `bin` 目录到系统PATH
- Linux: `sudo apt install g++`
- macOS: `xcode-select --install`

### Q2: 运行时"无法保存职员数据!"

**原因**: `data/` 目录不存在或无写入权限。
**解决**: 在程序根目录手动创建 `data` 文件夹。

### Q3: 中文显示乱码

**原因**: 终端编码设置问题。
**解决**:
- Windows: 在CMD中运行 `chcp 65001` 切换到UTF-8
- 或使用 Windows Terminal / PowerShell 7
- Linux/macOS: 确保终端支持UTF-8

### Q4: 如何重置管理员密码？

直接编辑 `data/employee.txt`，找到管理员行（E001），修改第三个字段（密码），重启程序即可。

### Q5: 数据文件被误删了怎么办？

删除 `data/` 目录下所有文件，重新运行程序，系统会自动创建默认数据。

### Q6: Web端和C++端的数据互通吗？

不互通。C++端使用 `data/employee.txt` 等管道分隔符文件，Web端使用 `data/account.txt` 格式。两端独立运行。

### Q7: 身份证号校验不通过？

中国18位身份证号需满足：
- 前17位为数字
- 第18位为数字或X
- 出生日期合法（1900-2100年间）
- 校验位正确（ISO 7064 MOD 11-2）

### Q8: 生物特征录入后如何删除？

生物特征存储在 `data/customer.txt` 第13个字段，可手动编辑文件删除，或通过 [A.客户证件信息补录] 清空。
